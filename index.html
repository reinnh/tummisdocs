Project Setup

To clone and set up the project locally:

    Step 1: Accept the GitHub Invitation

    Open your email and check for an invitation from GitHub. Click on the "View Invitation" link in the email. This will redirect you to GitHub, where you should click "Accept Invitation".

    Step 2: Set Up Your GitHub Environment

    Before working on the project, ensure you have: A GitHub account (sign up if you don‚Äôt have one). Git installed on your PC (if not, download it from [Git's official site](https://git-scm.com/)). A code editor like VS Code. Node.js (if the project is a JavaScript-based app) or any required software based on the tech stack.

    Step 3: Clone the Repository

    Open Git Bash or the VS Code terminal. Navigate to the folder where you want to store the project using:cd path/to/your/folder Clone the repository using:git clone You can find the repository URL on the project‚Äôs GitHub page under the Code button.

    Step 4: Set Up the Project Locally

    Navigate into the project directory: cd school-management-system Install dependencies (if applicable): If it‚Äôs a Node.js project, run: npm install

    Step 5: Create a New Branch

    To avoid modifying the main code directly, create a new branch for your work: git checkout -b your-branch-name Example: git checkout -b add-student-feature

    Step 6: Make Changes & Commit

    Open the project in VS Code and start making your changes. After making changes, check modified files: git status Add the changes: git add . Commit your changes with a message: git commit -m "Added student registration feature"

    Step 7: Push Your Changes to GitHub

    Push your changes to your branch: git push origin your-branch-name Go to GitHub and create a Pull Request (PR) from your branch to the main branch.

    Step 8: Collaborate & Merge

    Wait for feedback from the team. If changes are requested, modify the code and commit again. Once approved, your changes will be merged.

    Note: Stay Updated

    Since multiple people are working on the project: Fetch the latest changes before making new edits: git pull origin main Communicate with teammates via GitHub issues or a project chat.

GitHub Collaboration

To avoid conflicts while collaborating on GitHub:

    Always pull the latest changes before making edits: git pull origin main Create a new branch for your feature: git checkout -b feature-branch Make changes and commit with clear messages: git commit -m "Added feature X" Push your branch to GitHub: git push origin feature-branch Create a pull request and request review.

Deployment

The main project is hosted on Netlify at tummis.netlify.app

To deploy your version:

    Connect your GitHub repo to Netlify. Set build command: npm run build Deploy and get your custom link.

  • üìÇ src/ - The main source directory where all project files are located.
  • üìÇ components/ - Contains reusable UI components that are used across the application.
    • üìÇ common/ - Houses shared layout components:
      • Navbar.jsx - A reusable navigation bar that manages routing.

    • üìÇ forms/ - Manages all form-related UI components:
